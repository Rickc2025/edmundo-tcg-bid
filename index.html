Macau MVP payments

Simple now, scalable later

This prototype uses the realistic MVP flow: auction closes → buyer receives invoice summary → buyer pays by local bank transfer/manual transfer reference → admin confirms payment before release or shipping.

1. Win the lot Invoice includes lot total, buyer premium placeholder, shipping/pickup choice, and payment deadline.
2. Transfer payment Buyer uploads a bank transfer slip or shares transfer reference. Auto-payment gateway can be added later.
3. Staff verifies manually Admin marks payment confirmed, then releases pickup details or shipping fulfillment.

Trust & admin controls

  • Seller review required before lots go live
  • High-value lots can require extra identity proof
  • Manual approval flags for suspicious bidding patterns
  • Pickup or delivery only after payment confirmation
  • Issue reporting window clearly stated on each order
